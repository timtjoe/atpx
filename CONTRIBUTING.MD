# Contributing

First off, thank you for taking the time to contribute! Contributions are what make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct
By participating in this project, you are expected to uphold our commitment to a respectful and inclusive community.

## How Can I Contribute?

### Reporting Bugs
* Use the [GitHub Issues](https://github.com/timtjoe/atpx/issues) tab.
* Describe the bug, include steps to reproduce, and attach screenshots if possible.

### Suggesting Enhancements
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to  APTX.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. Ensure your code passes linting and type checks: `bun run type:check`.
3. Use **Conventional Commits** (e.g., `feat:`, `fix:`, `perf:`).
4. Open a Pull Request with a clear description of the changes.

## Commit Message Format
I follow the Conventional Commits specification:
* `feat`: A new feature for the user.
* `fix`: A bug fix for the user.
* `perf`: Code changes that improve performance.
* `refactor`: A code change that neither fixes a bug nor adds a feature.
* `docs`: Documentation only changes.
